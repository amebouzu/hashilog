## 1. エグゼクティブサマリー

走ログは、**日本のサーキットで刻んだラップタイムを、愛車情報と一緒に記録・共有できる Web サービス**です。

これまで「ガレージのホワイトボード」「個人ブログ」「SNS のスクショ」「LINE グループ」など、走行データはバラバラに散在していました。走ログはそれを **検索可能・比較可能・SNS 拡散可能** な形で集約し、ドライバー・サーキット・メーカーをつなぐプラットフォームを目指します。

### 提供価値

- **ドライバー**: 自分のタイム履歴・改造履歴・タイヤ履歴を1か所で管理。SNS で簡単にシェア。同条件ドライバーとの比較。
- **サーキット運営者**: 自施設の公式ページを無料で運用。走行会・イベントの告知。来場ドライバーのコミュニティ可視化。
- **タイヤ・パーツメーカー**: 自社製品でどのサーキット・どの車両でどのタイムが出ているかを実データで把握。
- **イベント主催者**: 走行会の集客チャネル。

### 現在のステータス

- 本番稼働中: https://hashilog.jp
- 主要機能はすべて実装済み（後述）
- インフラ: Vercel + Supabase Tokyo + GitHub
- 提携・データ提供等の問い合わせ受付中

## 2. 市場背景・課題認識

### 2.1 国内サーキット走行人口

- 国内のサーキット（ミニサーキット含む）は **30 か所以上**
- 走行会・タイムアタック・耐久などのアマチュアモータースポーツ参加層は数万人規模
- ドライバーの平均年齢層は 30〜50 代が中心、可処分所得が比較的高い

### 2.2 既存の情報インフラの課題

| 既存の手段 | 課題 |
|---|---|
| 個人ブログ・X / Instagram | 投稿が散在、検索性ゼロ、過去データに辿り着けない |
| LINE グループ・Discord | 閉じたコミュニティ、外部からは見えない |
| サーキット公式タイム掲示板 | サーキット間で形式バラバラ、車両・タイヤ情報が紐づかない |
| 走行会主催者の Excel | 主催者依存、走行会単位で完結、横断比較不可 |

### 2.3 走ログが解決すること

- **検索可能なフォーマット**: サーキット × 車種 × タイヤ × タイム のクロス絞り込み
- **エビデンス文化**: 計測機器・モニター写真の添付で記録の信頼性を担保
- **オープンな共有**: SNS 連携で X / Facebook / Instagram / Threads / LINE / リンクコピー まで網羅
- **公式の運営者ページ**: サーキット側がスタッフアカウントで自施設情報・告知を更新可能

## 3. サービス概要

### 3.1 サービスの基本フロー

1. ドライバーが走ログにサインアップ（メール+パスワード、無料）
2. 「マイガレージ」に愛車を登録（メーカー・モデル・改造内容・写真）
3. 「タイム投稿」でサーキット・タイム・タイヤ・天候等を記録
4. 投稿は自動でランキングに反映、SNS でシェア可能
5. サーキット運営者は自施設ページで来場ドライバーのタイムを把握、走行会告知が可能

### 3.2 主要画面（一例）

| 画面 | 役割 |
|---|---|
| トップ | サービス紹介・最新タイム・サーキット紹介への導線 |
| ランキング | サーキット × 車種 × タイヤで絞り込めるタイム順位表 |
| サーキット一覧 / 詳細 | 全国の登録サーキット紹介、施設ごとのタイムランキング、イベント告知 |
| マイガレージ | 自分の愛車一覧と改造内容、その車両のベストラップ |
| タイム投稿 | セクタータイム・最高速・天候・路面・気温・路温・タイヤ前後別銘柄まで記録 |
| ユーザーページ `/u/:username` | プロフィール、ベストラップ一覧、SNS リンク |
| アカウント設定 | プロフィール編集、SNS リンク登録、退会 |

## 4. 実装済み機能仕様（2026年5月時点）

走ログは「設計だけ」ではなく **すでに本番稼働している** プロダクトです。以下は実装完了済みの機能です。

### 4.1 ユーザー機能

- **アカウント**: メール+パスワード認証、確認メール、パスワードリセット、退会（GDPR/個人情報保護法対応）
- **プロフィール**: ユーザー名 / 表示名 / 自己紹介 / 都道府県 / アバター画像
- **SNS リンク**: X (Twitter) / Instagram / Threads / Facebook / YouTube / TikTok / Web サイト
- **退会**: アカウント完全削除（関連データもカスケード削除）

### 4.2 愛車（マイガレージ）

- 複数台の登録に対応
- メーカー / モデル / 年式 / カラー / 馬力 / 重量 / 自由記述説明
- **改造内容**: 足回り / エンジン / 駆動系 / ブレーキ / 外装 / 内装 を分野ごとに記述
- 写真ギャラリー（カバー写真 + 複数枚）

### 4.3 タイム投稿（中核機能）

- **タイム**: 総合タイム、セクター 1〜4 個別タイム（サーキットの設定セクター数に追従）
- **コンディション**: 天候（晴れ/曇り/雨/大雨/雪/変わりやすい）、路面（ドライ/ハーフウェット/ウェット）、気温、路温、最高速
- **タイヤ**:
  - ブランド × 銘柄 をドロップダウン選択 + 「その他（手動入力）」フォールバック
  - **前後で別銘柄に対応**（チェックボックスで切り替え）
  - サイズも前後別記録可能（例: F:245/40R18、R:265/35R18）
  - ユーザーが手動入力した銘柄は自動でブランドリストに追加され、次回以降ドロップダウン化
- **エビデンス写真**: 計測機器・モニター・車載動画キャプチャ等を複数枚添付可能
- **走行日・メモ**: 走行会名、アタック何回目、感想等を自由記述
- **編集・削除**: 投稿後も自分の投稿に限り編集・削除可能（履歴保持）

### 4.4 ランキング・検索

- **多次元フィルター**: サーキット × メーカー × 車種 × タイヤブランド × タイヤ銘柄 で絞り込み
- **ベストタイム表示**: 各車両ごとのベストラップ
- **モバイル最適化**: PC/スマホそれぞれに最適化されたレイアウト
- **最新投稿スライダー**: トップ・ランキングページに最新タイムをマーキー表示

### 4.5 サーキット情報

- 全国 25+ サーキットを既に登録済み
  - 国際サーキット: 富士、鈴鹿、もてぎ、岡山、オートポリス、SUGO、十勝、袖ヶ浦
  - ミニ・ショート: 筑波 2000/1000、セントラル、エビス東/西、日光、本庄、美浜、幸田、茂原西/東、スパ西浦、作手、中山、富士ショート、鈴鹿ツイン、スポーツランドやまなし
- サーキットごとに: 全長、セクター数、所在地、代表コーナー、特徴、公式 URL、紹介文
- サーキット詳細ページにそのコースのタイムランキング、走行会告知を集約

### 4.6 サーキット運営者向け機能（B2B 機能）

走ログの大きな差別化ポイントです。

- **公式アカウント**: サーキット運営会社のスタッフ用アカウント（個別付与）
- **自施設ページの編集**: 説明文・代表コーナー・特徴・公式 URL・公開状態を運営者自身が更新可能
- **イベント・告知の発行**:
  - 走行会、レース、お知らせ、固定スケジュール（毎週木曜 等）
  - タイトル / 本文 / 開始終了日時 / 表示用ラベル / 外部 URL / 公開状態
- 運営者には「このサーキットを編集」ボタンが自動表示

### 4.7 シェア・SNS 連携

- すべてのタイム詳細ページに 1-click シェアボタン
- 対応: X (Twitter) / Facebook / Instagram / Threads / LINE / リンクコピー
- OGP/Twitter Card 対応により SNS で美しくプレビュー表示
- JSON-LD（Schema.org）構造化データ実装済み（SportsEvent / Place / Organization）

### 4.8 SEO・公開チャネル

- メタデータ最適化（title / description / OG / Twitter Card）
- `sitemap.xml` 自動生成（サーキット・ユーザー・ラップ全件）
- `robots.txt` 設定済み
- 構造化データで Google リッチリザルト対応

### 4.9 通報・モデレーション

- 任意の投稿（タイム / 愛車 / プロフィール）に対する通報機能
- 運営側で内容確認・対応のフロー実装済み

### 4.10 信頼性・運用

- **インフラ**: Vercel（Next.js 14 SSR）+ Supabase Tokyo（Postgres + Storage + Auth）
- **死活監視**: `/api/health` エンドポイント（DB 接続も含めた疎通確認）
- **アクセス解析**: Cloudflare Web Analytics（Cookie 不要）対応 / 環境変数でON/OFF
- **PWA 対応**: スマホで「ホーム画面に追加」してアプリ風利用が可能
- **個人情報保護**: 利用規約 / プライバシーポリシー / 特定商取引法表記 すべて整備済み
- **法令遵守**: 暗号化通信、Supabase Tokyo（国内データ保管）、退会時の完全削除

## 7. 競合・差別化

### 7.1 既存の選択肢との比較

| 項目 | 既存ブログ | SNS | サーキット公式 | **走ログ** |
|---|---|---|---|---|
| 検索性 | 低 | 中 | 中 | **高（多次元フィルター）** |
| データ構造化 | × | × | △ | **○** |
| エビデンス | △ | △ | ○ | **○（写真添付）** |
| 横断比較 | × | × | × | **○** |
| サーキット公式情報統合 | × | × | ○（自社のみ） | **○（横断）** |
| SNS シェア | △ | ○ | × | **○（OGP 最適化）** |
| 改造内容との紐付け | △ | △ | × | **○** |
| **タイヤ前後別記録** | × | × | × | **○** |

### 7.2 走ログの強み

1. **データ構造の妥協のなさ**: タイヤ前後別、セクター 4 個別、気温・路温・最高速まで保存
2. **B2B 機能内蔵**: サーキット運営者・将来的にメーカーが管理コンソールを持てる
3. **国内データ保管**: Supabase Tokyo によりデータは日本国内のみで処理
4. **オープンな思想**: 中央集権的なギルドではなく、SNS のように個人投稿の集合体
5. **モバイルファースト**: スマホでの記録・閲覧・シェアを前提にした UI

## 9. データ・分析価値

走ログは「ユーザー投稿型のサーキット走行データベース」として、以下の集計データを保有・蓄積しています：

- サーキット別の **車種シェア・タイヤシェア・年代シェア**
- 季節別・天候別のタイム分布
- 同一車種における改造内容とタイムの相関
- タイヤ銘柄ごとの使用率・好まれるサーキット
- ユーザーの SNS 連携率、ピーク時間帯、地域分布

これらは個人を特定しない集計形式で、メーカー・媒体・サーキット運営者にレポートとして提供可能です。
